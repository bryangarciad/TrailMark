Setup & Deployment

Create the Xcode project from scratch, add the watch app to the existing iOS project, share the TrailmarkCore package, wire permissions and capabilities, then run on the Simulator and on real devices.

Step 0

Prerequisites

  • Xcode 26 or later (this project targets iOS / watchOS 26).
  • A free Apple ID is enough for Simulator + 7-day device testing; a paid Apple Developer account avoids the weekly re-sign.
  • For the wrist-only features (heart rate, Core Motion, real workouts, true sync): a physical iPhone + paired Apple Watch. The Simulator can't produce sensor data.
  • Decide a bundle id prefix you own, e.g. com.yourname. This guide uses ramsesg.TrailMark.

Step 1 · Course 1.1

Create the iOS project

The iPhone app is created first; the watch and the shared package are added to it afterwards.

  1. New project Xcode → File → New → Project… → iOS tab → App → Next.
  2. Name & options Product Name TrailMark. Interface SwiftUI, Language Swift, Storage None, Testing System None (or XCTest). Set your Team and an Organization Identifier (e.g. ramsesg, giving bundle id ramsesg.TrailMark).
  3. Save Choose your repo folder. Leave "Create Git repository" checked if you want version control. You now have TrailMarkApp.swift + ContentView.swift.
  4. Confirm it runs Pick an iPhone simulator and press ⌘R — you should see "Hello, world!". That confirms signing + toolchain before you add complexity.
Modern project layout Xcode 26 projects use file-system-synchronized groups — any .swift file you drop into the target's folder is compiled automatically. You will not need to manually add the feature files to the target; just create them inside the app folder.

Step 2 · Course 2.1

Add the watchOS app to the existing iOS project

This is the key "don't start a second project" move. The watch app is a target inside the same project, paired to the iPhone app.

  1. New target With the project open: File → New → Target… → watchOS tab → App → Next.
  2. Pair it to the iPhone app Product Name TrailMarkCompanion Watch App. Crucially set "Watch App for an Existing iOS App" via the Companion iOS App dropdown → choose TrailMark. Interface SwiftUI. This sets WKCompanionAppBundleIdentifier so the two apps install together.
  3. Activate the scheme When Xcode offers to activate the new scheme, click Activate.
  4. Verify the bundle ids nest The watch id should be the iPhone id plus a suffix, e.g. iPhone ramsesg.TrailMark → watch ramsesg.TrailMark.watchkitapp. The iOS target also gains an Embed Watch Content build phase automatically.
  5. Run the watch scheme Select the watch scheme + a paired Apple Watch simulator → ⌘R. Xcode boots the iPhone + Watch sim pair together.
Watch out Pick "Watch App for an Existing iOS App", not the standalone "App" with no companion. The companion link is what lets WatchConnectivity (Course 3.1) and "install through the phone" work.

Step 3 · Course 1.1, reused in 2.1

Create the shared TrailmarkCore package

A single local Swift package holds every model and manager. Both apps link it, so logic is written once.

3a · Create the package

  1. New package File → New → Package… (or File → New → Project → Multiplatform → Swift Package). Name it TrailmarkCore.
  2. Save it inside the project folder Put it next to the .xcodeproj (e.g. TrailMark/TrailmarkCore) so it travels with the repo. If Xcode offers "Add to: TrailMark", accept — that adds it as a local package in one step.
  3. Set platforms In Package.swift, declare the platforms so the iOS/watch APIs resolve: 
    platforms: [ .iOS("17.0"), .watchOS("10.0") ],
  4. Add the source folders Create Sources/TrailmarkCore/ subfolders: Models, Health, Media, Location, Motion, Connectivity, Workout, Persistence, Support — see the Build Guide for what goes where, class by class.

3b · Link the package to both app targets

  1. Add local package (if not already) Project file selected → Package Dependencies tab → + → Add Local… → choose the TrailmarkCore folder.
  2. Link to iOS target Select the TrailMark target → General → Frameworks, Libraries, and Embedded Content → + → add the TrailmarkCore library.
  3. Link to watch target Repeat for the TrailMarkCompanion Watch App target.
  4. Import & verify In any view add import TrailmarkCore. It should autocomplete. Build both schemes.
Platform-specific code lives in the package Use #if os(watchOS) / #if canImport(UIKit) guards inside the package for the few APIs that differ (e.g. HKWorkoutSession is watchOS-only; video thumbnails are iOS-only). The build guide flags each one.

Step 4

Permissions — Info.plist usage strings

Both apps use GENERATE_INFOPLIST_FILE = YES, so there's no Info.plist to edit directly. Add these in the target's Info tab (Custom iOS/watchOS Target Properties → +), using the raw key names below.

Silent failure if missing Without a usage string the system silently denies the request — for HealthKit the permission sheet won't even appear. If a feature seems to do nothing, check here first.

iOS target — TrailMark

KeySuggested value
NSHealthShareUsageDescriptionTrailMark reads your steps, distance, energy, heart rate and sleep to show your daily activity and recovery.
NSHealthUpdateUsageDescriptionTrailMark saves your recorded activities to the Health app as workouts.
NSLocationWhenInUseUsageDescriptionTrailMark records the route of your journey and geotags the memos you capture.
NSMicrophoneUsageDescriptionTrailMark records voice memos for your field journal.
NSCameraUsageDescriptionTrailMark records short video memos for your field journal.
NSPhotoLibraryUsageDescriptionUsed to pick a video on the Simulator, which has no camera. (Optional — device-only flow doesn't need it.)
NSMotionUsageDescriptionTrailMark reads motion data to detect your cadence and activity.

watchOS target — TrailMarkCompanion Watch App

KeySuggested value
NSHealthShareUsageDescriptionTrailMark reads your heart rate, steps and energy on your wrist.
NSHealthUpdateUsageDescriptionTrailMark saves your workouts to the Health app.
NSMicrophoneUsageDescriptionTrailMark records voice memos on your wrist.
NSMotionUsageDescriptionTrailMark reads motion data to detect your cadence and activity.

Step 5

Capabilities

For each target: Signing & Capabilities → + Capability.

iOS — TrailMark

  • HealthKit — required for the Health usage keys to take effect.

watchOS — Watch App

  • HealthKit.
  • Background Modes → Workout Processing — lets the live workout session (Course 3.2) keep running while backgrounded. Adds WKBackgroundModes = [workout-processing].

All three (complication only)

  • App Groups with the same id on iOS app, watch app, and the complication target:
group.ramsesg.TrailMark

The code reads/writes today.steps here. Until the group exists the complication shows a sample number — it still builds.

Signing

Keep Automatic signing and set your Team on every target. If you see "Failed to register bundle identifier", change the ids to something globally unique (prefix with your own reverse-domain).

Step 6 · Course 3.4

Create the complication target

App-extension targets are created from a template, not by editing the project file. The source is already written at TrailMark/TrailMarkComplication/TrailMarkComplication.swift.

  1. New target File → New → Target… → watchOS → Widget Extension.
  2. Name & embed Name it TrailMarkComplication, uncheck "Include Configuration App Intent", and ensure the embedding app is the Watch App.
  3. Swap in the source Replace the generated file's contents with the prewritten TrailMarkComplication.swift.
  4. App Group Add the App Groups capability (group.ramsesg.TrailMark) to this target so it can read live steps.
  5. Add to the face Build the watch scheme. On the watch, long-press the face → Edit → add the TrailMark complication, or add it to the Smart Stack.

Step 7

Running on the Simulator

FeatureSimulator?Notes
UI, navigation, charts, persistenceYesFully works.
HealthKit reads (steps/sleep/energy)PartialOnly if you add sample data in the Simulator's Health app.
HealthKit write (save workout)YesWrites into the simulated Health store.
Voice memo (audio)YesUses the Mac microphone.
Video memoPartialNo camera → the code falls back to the photo library.
Location / routeYesUse Features → Location → Freeway Drive to simulate a moving track.
Live heart rate / Core MotionNoNo sensors — values stay at 0. Expected.
Live workout sessionPartialStarts/saves, but metrics won't stream.
WatchConnectivity syncYesWorks between a paired iPhone + Watch sim pair.

Add Health sample data

  1. Authorize Run the iOS app once and accept the Health prompts.
  2. Add data Open the Health app inside the same Simulator → Browse → e.g. Steps → Add Data. Repeat for Distance, Active Energy, Sleep. Re-open TrailMark.

Run from Xcode

  1. iPhone Scheme TrailMark → an iPhone simulator (e.g. iPhone 17) → ⌘R.
  2. Watch Scheme TrailMarkCompanion Watch App → a paired Apple Watch sim → ⌘R. Pairing is automatic; check via Simulator → File → Manage Paired Devices.

Run from the command line

open -a Simulator
xcodebuild -project TrailMark/TrailMark.xcodeproj -scheme TrailMark \
  -destination 'platform=iOS Simulator,name=iPhone 17' build

xcrun simctl boot "iPhone 17" 2>/dev/null || true
xcrun simctl install booted "$(xcodebuild -project TrailMark/TrailMark.xcodeproj \
  -scheme TrailMark -destination 'platform=iOS Simulator,name=iPhone 17' \
  -showBuildSettings | awk -F' = ' '/ BUILT_PRODUCTS_DIR /{d=$2} / FULL_PRODUCT_NAME /{p=$2} END{print d"/"p}')"
xcrun simctl launch booted ramsesg.TrailMark

Step 8

Running on real devices

Heart rate, Core Motion, real workouts and true watch↔phone sync only work on hardware — exactly the things the curriculum says can't be simulated.

One-time device prep

  1. Developer Mode On iPhone & Watch: Settings → Privacy & Security → Developer Mode → On (the device reboots). On the Watch this appears after connecting to Xcode once.
  2. Cable the phone The Watch installs through the paired iPhone — you don't cable the Watch. Confirm the Watch is paired to that iPhone.
  3. Trust In Xcode: Window → Devices and Simulators → confirm both appear.

Run

  1. iPhone TrailMark scheme → your iPhone → ⌘R. Approve Health/Location/Mic/Camera/Motion prompts.
  2. Watch Watch App scheme → your Apple Watch → ⌘R (first install can take minutes). Approve Health/Mic/Motion on the watch.
  3. Trust certificate First device launch: Settings → General → VPN & Device Management → trust your developer certificate.
Verifying the hard-to-test features Workout (1.3 / 3.2): after finishing, Health app → Browse → Workouts.   Pocket sync (3.1): record on the watch → it appears in the iPhone Journal/Journeys within seconds (foreground) or after the phone next wakes.   Complication (3.4): add to the face; refreshes ~every 15 min.

Step 9

Build sanity checks

# Shared package, both platforms (run inside TrailMark/TrailmarkCore)
xcodebuild build -scheme TrailmarkCore -destination 'generic/platform=iOS'
xcodebuild build -scheme TrailmarkCore -destination 'generic/platform=watchOS'

# Apps
xcodebuild build -project TrailMark/TrailMark.xcodeproj -scheme TrailMark \
  -destination 'platform=iOS Simulator,name=iPhone 17'
xcodebuild build -project TrailMark/TrailMark.xcodeproj -scheme "TrailMarkCompanion Watch App" \
  -destination 'platform=watchOS Simulator,name=Apple Watch Series 11 (46mm)'

Step 11 · optional

Publish this site to GitHub Pages

A workflow at .github/workflows/deploy-pages.yml publishes the Docs/ folder on every push to main.

  1. Enable Pages Repo on GitHub → Settings → Pages → Build and deployment → Source: GitHub Actions.
  2. Push Commit and push to main. The Action runs and prints the published URL (typically https://<user>.github.io/<repo>/).
  3. Share Share that URL with anyone — all four pages are cross-linked from the top nav.
This makes the material public GitHub Pages serves to the open web. Don't include anything private (student data, unreleased grades). The course content here is fine to publish.
